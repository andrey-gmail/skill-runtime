# NOTES

## 1. В чём единый формат уступает?

Формат уступает для **csv-explore**. Он знает только два состояния: полная свобода (нет `steps`) или жёсткий порядок (есть `steps`). Промежуточного варианта — рекомендованная последовательность без принуждения — нет. Для исследовательского skill'а было бы полезно задать мягкие подсказки вроде "обычно стоит начать со структуры, потом аномалии", которые рантайм инжектирует, но не навязывает. Сейчас это можно сделать только через `system_prompt`, но рантайм никак это не структурирует — всё ложится на промпт-инженерию.

## 2. Что случится при нарушении порядка шагов?

Модель физически не может поменять порядок или пропустить шаг — она не знает о существовании следующих шагов. Рантайм инжектирует промпт каждого шага отдельно и переходит к следующему только программно, когда модель завершила текущий. Нет механизма, через который модель могла бы "попросить" перейти к другому шагу.

Честное ограничение: модель может в рамках одного шага написать Python-код, который выполнит все операции разом. Рантайм контролирует порядок промптов, но не семантику исполняемого кода — это фундаментальное ограничение без формальной верификации.

## 3. С чем не согласен?

С двумя вещами. Во-первых, `run_python` без песочницы — произвольный Python с доступом к файловой системе и сети это осознанный долг даже для минимального рантайма. Таймаут и ограничение cwd уже есть, но Docker или `--no-site-packages` были бы следующим шагом. Во-вторых, условие завершения шага ("модель вернула текст без tool_calls") архитектурно хрупкое — модель может ответить "Понял, начинаю" и рантайм посчитает шаг выполненным. На практике это решается качеством промпта, но не архитектурой.
